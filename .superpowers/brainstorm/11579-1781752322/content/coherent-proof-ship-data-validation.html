Design Section 3: Data Model, Loader Boundaries, and Validation Gates

This section defines how the curated ship becomes reliable data, how the Godot loader consumes it, and what must fail before we accept it.

Golden fixture data path and validation gates

Data model decisions

The curated proof ship gets its own named fixture files rather than modifying seed 17 in place. It should still use the current kit and loader path, but the fixture may add explicit metadata for concepts the current generated fixture only implies poorly: doors, blocked doors/routes, landmarks, and vertical transitions.

Component boundaries

  • Fixture data says what the ship is: rooms, deck heights, portals, blockers, landmarks, objective markers.
  • GeneratedShipLoader turns that data into structural nodes and queryable markers. It does not decide game progression.
  • PlayableGeneratedShip wires player, camera, objectives, and prompts onto the loaded ship.
  • Validation scripts prove the ship is connected, readable, and walkable before we accept it.

Minimum validation gates

  • Topology: every room reachable from the airlock through declared doors/transitions.
  • Spatial continuity: the central spine connects branches without disconnected box islands.
  • Vertical clarity: deck transition has both data metadata and physical/visible runtime representation.
  • Blocker semantics: blocked route is visible and not accidentally traversable.
  • Player proof: player can traverse from airlock to reactor destination in the playable scene.
  • Visual proof: fresh Godot capture shows airlock/spine/transition/destination or enough of them to prove readability.
A

Approve data + validation design

Use named golden fixture files, minimal schema additions, existing loader path, and explicit spatial/readability validation gates.

B

Keep schema stricter

Avoid new fields; express everything using existing layout/gameplay fields even if less direct.

C

Broaden schema now

Define a richer ship DSL immediately for rooms, portals, hazards, landmarks, and gameplay beats.
