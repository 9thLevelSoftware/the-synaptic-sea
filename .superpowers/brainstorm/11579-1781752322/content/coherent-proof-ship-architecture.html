Design Section 1: Coherent Proof Ship Architecture

This section defines how the curated ship should be represented so it becomes part of the real game pipeline, not a one-off diorama.

Recommended architecture: hand-designed intent → data fixture → existing Godot loader → playable proof

Design decision

The proof ship should be data-authored, not hand-only. We intentionally design the rooms, then encode them as a canonical fixture consumed by the same Godot runtime path. If the current schema cannot express a needed concept, we add the smallest explicit field for that concept.

A

Approve this architecture

Proceed with a data-driven golden fixture using the existing loader, plus minimal schema additions for doors/decks/blockers only where necessary.

B

Prefer a hand-authored Godot scene

Prioritize fast visual/spatial feel even if it temporarily bypasses the generated-data path.

C

Prefer generator-first

Invest in generation rules immediately, even if the next visible proof takes longer.
